OVERVIEW
--------
Built a fully isolated "sandbox" clone of the entire website inside the /test/
subfolder, reachable at allprobuildingsupplies.com/test/. It is a functional
copy of all 10 pages but uses placeholder URLs for the Cloudflare Worker and
EmailJS, so it cannot read or write production data or send real emails.

The refactor combines Tier 1 (deduplication / shared partials / shared helpers)
and Tier 2 (full ES-module architecture with component rendering, event
delegation, and a sandbox-safe API layer).

--------------------------------------------------------------------------------
SANDBOX ISOLATION (CRITICAL)
--------------------------------------------------------------------------------
config.js sets:
  WORKER_URL   = "about:blank"
  EMAILJS_PUB  = "SANDBOX_PUBLIC_KEY"
  EMAILJS_SVC  = "SANDBOX_SERVICE"
  EMAILJS_TPL  = "SANDBOX_TEMPLATE"
  ADMIN_PIN    = "sandbox"          (test-only PIN for admin access)
  isSandbox    = true

api.mjs behavior in sandbox:
  - loadProducts / loadUsers / loadOrders return [] (or empty)
  - saveProductsCSV / saveUsers / saveOrders are no-ops that resolve true
  - NO network calls to the live Cloudflare Worker

email.mjs behavior in sandbox:
  - initEmail() skips emailjs.init if placeholder keys detected
  - sendEmail() logs payload to console and resolves with { sandbox: true }
  - NO emails ever sent

ui.mjs injects a fixed-top "TEST SANDBOX - no live data" banner on every page
so testers can never confuse it with production.

--------------------------------------------------------------------------------
KEY REFACTOR WINS
--------------------------------------------------------------------------------
1. HTML DEDUPLICATION
   - 7 shared partials replace ~150 lines of duplicated markup per page
   - Every page uses data-include for cursor, topbar, nav, mobile menu,
     ticker, cta-band, and footer
   - include.mjs loads them once, then fires apbs:ready so page scripts
     can run against the fully assembled DOM

2. CSS CONSOLIDATION
   - All inline <style> blocks from 10 HTML pages merged into one
     stylesheet
   - Deduped variables, button styles, form styles, table styles,
     modal styles, hero styles
   - Single source of truth for design tokens in :root

3. JS MODULARIZATION
   - 8 shared ES modules + 4 components + 10 page scripts
   - Replaced inline <script> blocks in every page
   - Event delegation replaces all onclick="..." attributes
   - Each page script imports only what it needs

4. DATA LAYER
   - api.mjs is the single gateway to GitHub/Worker CSV/JSON reads/writes
   - All pages now use the same parseCSV / stringifyCSV helpers
   - Cart logic centralized in cart.mjs (localStorage key: apbs_cart)
   - Auth logic centralized in auth.mjs (sessionStorage: apbs_user,
     localStorage: apbs_admin_auth)

5. EMAIL TEMPLATE DEDUPE
   - Three near-identical ~300-line inline HTML templates in admin.html
     collapsed into composable fragments in email.mjs
   - buildConfirm / buildDelivered / buildInvoice share header / footer /
     line-item table / totals fragments

6. BUG FIXES
   - login.html: plaintext password was being compared to a stored SHA-256
     hash. auth.mjs now hashes the input before compare, with a graceful
     legacy-plaintext fallback.
   - cart data key mismatch: unified on item.unitPrice and item.pcsPerCtn
     across cart.mjs and cart-row.mjs

DEPLOYMENT
--------------------------------------------------------------------------------
1. Upload the entire /test/ folder to the site root
2. Access at: https://allprobuildingsupplies.com/test/
3. Admin dashboard PIN: "sandbox" (change in assets/config.js if needed)
4. Nothing in /test/ touches production:
     - No writes to the GitHub proxy
     - No real emails via EmailJS
     - Separate localStorage namespace is not enforced (keys are shared),
       so if testers log in on /test/ they may stay logged in on the live
       site and vice versa. If full key isolation is needed, prefix all
       storage keys with "apbs_test_" in api.mjs / auth.mjs / cart.mjs.
